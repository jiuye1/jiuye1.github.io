新的时间记录方案

发表于 2015-06-07   |   分类于 NONSENSE

为什么要换方案呢？似乎有点太折腾。但是为什么不折腾呢？没有理由不折腾，所以还是折腾吧。

上个方案太依赖电脑，所有的表格笔记都需要打开笔记本，很麻烦。加之从周五开始到昨天学会了用Git做静态网页，于是这一个博客链接了新浪SAE、又有静态Git页，心里觉得怪怪的。但Git+hexo+markdown这种写博客的方式实在令人太喜欢，于是果断把新浪SAE丢掉了，虽然速度挺好。

便决定把常识性的笔记放进Github，GTD内容也直接用文章形式放在这个博客内。两个都喜欢，真是麻烦呐。 一直在想哪些步骤还可以不用电脑。因为手机是windows，应用少，没有markdown编辑器。比较完整的应用只有onenote和evernote在wp上。onenote自然是无比强大，可是那个直接导入文章的wordpress插件怎么整也没法子，遂找啊找啊，找到一个可以直接把印象笔记发表到博客的方法。

具体就是：eatags.com 注册一个用户，然后把evernote授权，最后绑定一个wordpress博客的用户（新建一个非管理员的用户，作者权限）。

用evernote新建笔记时，选择eat.wordpress.post标签，同步后，该笔记将会自动发布到wordpress博客上。如果不想马上发布文章，只想生成wordpress博客草稿，那就选择eat.wordpress.draft标签。 这个方法很棒，而且操作及其简单，试了一下，立刻看到博客里出现evernote的草稿了。

唯一需要注意的是，这个网站需要国际版印象笔记。注册一个就行了。 本来自己制作了一个evernote的日程记录模板，结果手机对模板的支持不太好。干脆直接从文字记录好了，然后直接形成草稿，简单方便。

晚上用半小时做一个大概规划，然后用@表示番茄；如果没有按照计划做，便用/隔开，写下做了什么以及番茄。番茄味道真心不错，番茄酱也很好吃。

最后赞美一个Git吧！打开速度真是太快了！本地编辑文档然后上传，真是充满了技术小能手的感觉！尤其是我现在使用的next主题，一级简洁，显示文章目录结构一级完美，真是爽呆！我在想，干脆把考试所有笔记全部写在一起，形成结构完整的一本书！！明天试试吧。

但论功能多样，Git搭起来的博客还是不如wordpress，毕竟我不会自己写。虽说wordpress相比之下太臃肿了，但是，wordpress的傻瓜操作又是如此亲民，当然Git是如此酷炫。所以，两个一起用吧。

怎么着竟有一种齐人之福的感觉，奇人实际上是很累吧。

对了，要把所有日程记录的文本都放进GTD的分类目录中，而且要GTD目录下的文章都不显示在主页。做完这个，我要整理精神好好玩玩儿啦！这个周末整个人几乎一直黏在键盘上，下个周末可得出去跑跑。

Github+hexo搭建静态博客

发表于 2015-06-06   |   分类于 SUMMARY

虽然Wordpress一直用着挺好，但自从知道Git的存在之后就心痒难耐。虽然一时并不非常需要，但是不搞明白就不安心，况且学会了总可以找到用途的。昨天通宵看了各路大神的教程，一遍一遍照做，却总是出现各种莫名问题，加之各个教程相似度既高又有区别，总之糊里糊涂到最后也没弄好。睡一觉醒了之后，突然想明白，其实是有三个大步骤。

第一步是用hexo在本地搭建博客，以能够打开本地网页为成功标志。第二步是建立本地与Github的联系，以SSH的配置成功为成功标志。第三步是将本地博客上传到Github仓库，以打开网页博客为成功标志。

之前在网上看教程，只是照搬步骤，却没弄懂各个步骤的本质，所以失败之后，排查问题犹如大海捞针。一旦想明白之后，各种不同的教程我也能有针对性的取其精要，并迅速找到问题，最后终于成功。

这种撒花的心情吖：http://jiuye1.github.io/！趁此时便把步骤记录下来，难保时间长忘记了。

## 配置Git仓库

### 注册GitHub

[http://www.github.com/](http://www.github.com/)

记住username和邮箱，注册完之后要去邮箱激活。然后建个仓库，跟着页面走就是。仓库名称必须是`yourname.github.io`形式，记着勾选`Initialize this repository with a README`<span class="Apple-converted-space"> </span>选项。

### 配置SSH keys

  1. 检查电脑上的SSH sky:

    cd ~/.ssh

  2. 生成新的SSH sky:

    ssh-keygen -t rsa -C "youremail@youremail.com"

    系统要求输入密码的时候直接输，不会有显示。

    显示出这个画面就说明好了：

    添加SSH Key到GitHub

  3. 在本地打开C:Documents and SettingsAdministrator.sshid_rsa.pub文件，复制。

  4. 登陆github，Account Settings—->SSH Public keys —-> add another public keys

  5. add key，粘贴。

    测试

    输入：

    ssh -T git@github.com

    要选yes or no的时候就打yes，最后看到

    `Hi cnfeat! You've successfully authenticated, but GitHub does not
provide shell access.
`

    OK了。

    完善用户信息

  6. <span class="pln">git config </span><span class="pun">--</span><span class="kwd">global</span><span class="pln"> user</span><span class="pun">.</span><span class="pln">name </span><span class="str">"cnfeat"</span><span class="com">//用户名 </span>

  7. <span class="pln">git config </span><span class="pun">--</span><span class="pln">globaluser</span><span class="pun">.</span><span class="pln">email </span><span class="str">"cnfeat@gmail.com"</span><span class="com">//自己的邮箱</span>

    将本地博客部署到Git

    配置_config.yml文件

    `deploy:
type: git
repository: git@github.com:youername/yourname.github.com.git
branch: master
`

    据说有的老版本type写github。

    完成部署

    `hexo clean
hexo generate
hexo deploy
`
    ### 本地查看调试 
    `hexo g #生成
hexo s #启动
`

    每次有更改都要hexo g 一下，hexo s后可以在本地查看。同步到仓库是：

    `hexo d
`

    因为我不绑域名费的，所以就不操心DNS设置啦。

    hexo常用操作

    更换主题

    网上有很多主题：直接 git clone 地址，类似下面这样：

    ` git clone https://github.com/A-limon/pacman.git themes/pacman
`

    安装成功后，在根目录文件夹下面的_config.yml，把里面的theme属性设置为pacman。然后进入themes/pacman，运行Git bash， 执行

    ` git pull
`

    进入根目录，运行

    ` hexo s
`

    然后在浏览器输入localhost:4000就能看到新主题了。

    写新文章

    输入：

    `hexo n “文章名”
`

    文章名字如果有空格一定要用引号。然后就会在source_post下面发现一个文章名.md的markdown格式的文件。用markdown打开该文件，编辑保存。

    ` hexo g && hexo s

本地就可以看到了。

或者直接用markdown编辑好文件，直接放进文件夹，然后执行同样的命令。注意格式。
