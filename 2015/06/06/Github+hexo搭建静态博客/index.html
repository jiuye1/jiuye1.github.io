Github+hexo搭建静态博客

发表于 2015-06-06   |   分类于 Web

昨天通宵看了各路大神的教程，总是出现各种莫名问题，糊里糊涂到最后也没弄好。睡一觉后，突然想明白，其实是有三个大步骤。

第一步是用hexo在本地搭建博客，以能够打开本地网页为成功标志。第二步是建立本地与Github的联系，以SSH的配置成功为成功标志。第三步是将本地博客上传到Github仓库，以打开网页博客为成功标志。

之前在网上看教程，只是照搬步骤，却没弄懂各个步骤的本质，所以失败之后，排查问题犹如大海捞针。一旦想明白之后，各种不同的教程我也能有针对性的取其精要，并迅速找到问题，最后终于成功。

这种撒花的心情吖：http://jiuye1.github.io/！趁此时便把步骤记录下来，难保时间长忘记了。

准备软件

依次安装：
• 1、Node.js
• 2、Git

安装好之后，后面最常用的操作是：鼠标右键—打开Git Bash。

搭建Hexo本地博客

安装Hexo

打开git，输入：

npm install hexo -g

部署Hexo

建立一个「Hexo」的文件夹，在此文件夹中右键打开Git Bash,然后安装几个必要的插件：
EJS

npm install hexo-renderer-ejs --save

Stylus

npm install hexo-renderer-stylus --save

Marked

npm install hexo-renderer-marked --save

接着建立博客：

hexo init

发布：

hexo generate && hexo server

如果成功了，在浏览器中输入localhost:4000便能看到博客，失败的话注意看提示，也能解决。

配置Git仓库

注册GitHub

http://www.github.com/

记住username和邮箱，注册完之后要去邮箱激活。然后建个仓库，跟着页面走就是。仓库名称必须是yourname.github.io形式，记着勾选Initialize this repository with a README 选项。

配置SSH keys

  1. 检查电脑上的SSH sky:

    cd ~/.ssh

  2. 生成新的SSH sky:

    ssh-keygen -t rsa -C "youremail@youremail.com"

系统要求输入密码的时候直接输，不会有显示。

显示出这个画面就说明好了：

添加SSH Key到GitHub

  1. 在本地打开C:\Documents and Settings\Administrator.ssh\id_rsa.pub文件，复制。

  2. 登陆github，Account Settings—->SSH Public keys —-> add another public keys

  3. add key，粘贴。

测试

输入：

ssh -T git@github.com

要选yes or no的时候就打yes，最后看到

Hi cnfeat! You've successfully authenticated, but GitHub does not
provide shell access.

OK了。

完善用户信息

将本地博客部署到Git

配置_config.yml文件

deploy:
type: git
repository: git@github.com:youername/yourname.github.com.git
branch: master

据说有的老版本type写github。

完成部署

hexo clean
hexo generate
hexo deploy

本地查看调试

hexo g #生成
hexo s #启动

每次有更改都要hexo g 一下，hexo s后可以在本地查看。同步到仓库是：

hexo d

因为我不绑域名费的，所以就不操心DNS设置啦。

hexo常用操作

更换主题

网上有很多主题：直接 git clone 地址，类似下面这样：

git clone https://github.com/A-limon/pacman.git themes/pacman

安装成功后，在根目录文件夹下面的_config.yml，把里面的theme属性设置为pacman。然后进入themes/pacman，运行Git bash， 执行

git pull

进入根目录，运行

hexo s

然后在浏览器输入localhost:4000就能看到新主题了。

写新文章

输入：

hexo n “文章名”

文章名字如果有空格一定要用引号。然后就会在\source_post下面发现一个文章名.md的markdown格式的文件。用markdown打开该文件，编辑保存。

hexo g && hexo s

本地就可以看到了。

或者直接用markdown编辑好文件，直接放进文件夹，然后执行同样的命令。注意格式。

#Git
